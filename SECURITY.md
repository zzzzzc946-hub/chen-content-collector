# 安全策略

## 支持范围

仅维护当前公开版本的安全修复。不要在 Issue、Pull Request、日志、截图或提交信息中粘贴 Token、Cookie、密码、私钥、用户数据或可复用分享链接。

## 报告漏洞

在创建公开 GitHub 仓库前，维护者必须启用 GitHub 的 private vulnerability reporting，并配置受控的安全联络渠道。完成前不接受包含漏洞细节的公开报告。

仓库启用私密报告后，请通过该入口提交：

- 受影响版本与复现步骤；
- 影响范围；
- 建议修复方向（如有）。

报告中请使用脱敏示例。若意外提交了凭证，应立即在相应服务端撤销或轮换，并将该事件作为安全问题处理。

## 发布前门禁

维护者必须在 Release 前确认：

1. 公开导出包通过个人路径、个人网络端点、运行数据与凭证扫描；通用回环地址仅作为本地安全边界保留；
2. GitHub Actions 只从受控 secret store 读取生产凭证；
3. DMG、校验和与 manifest 均不含用户数据或本机配置；
4. 第三方依赖许可证已复核。
