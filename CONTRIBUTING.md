# 贡献指南

## 提交范围

每个分支只处理一个明确目标。提交前确认不包含 `.env`、`.dev.vars`、`config.json`、数据库、浏览器资料、构建产物、媒体、日志、个人路径或生产凭证。

## 开发流程

1. 从当前公开基线创建独立分支或 worktree。
2. 先为行为变化添加最小失败测试，再实现并验证。
3. 运行与变更范围匹配的测试、静态检查和格式检查。
4. 更新 `VERSION`、`CHANGELOG.md` 与 `docs/版本记录.md`（如变更影响发布版本）。
5. 不要在提交信息、测试 fixture 或文档中写入真实服务端点、个人网络端点、个人文件路径或凭证；通用回环地址可用于本地安全边界测试。

## Pull Request

Pull Request 应说明目的、测试证据、兼容性影响和回滚方式。涉及安全、权限、存储、部署或依赖许可证的变更，必须附带相应审计说明。
